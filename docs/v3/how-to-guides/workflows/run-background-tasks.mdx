### Start a background task worker

To start a worker to run background tasks for a single function, define a task and call `.serve()` on it.

{/* pmd-metadata: notest */}
```python
from prefect import task

@task(log_prints=True)
def add(a: int, b: int):
    print(f"{a} + {b} = {a + b}")

add.serve()
```

### Trigger a background task

To trigger a background task, call `.delay()` on the same task in a different terminal.

{/* pmd-metadata: notest */}
```python
add.delay(1, 2)
```

<Tip>
`.delay()` has the same signature as the `@task` decorated function.
</Tip>

### Start a worker to run background tasks for multiple functions

To start a worker to run background tasks for multiple functions, call `serve()` with the tasks you want to run.

{/* pmd-metadata: notest */}
```python
from prefect import task
from prefect.task_worker import serve

@task(log_prints=True)
def add(a: int, b: int):
    print(f"{a} + {b} = {a + b}")

@task(log_prints=True)
def multiply(a: int, b: int):
    print(f"{a} * {b} = {a * b}")

# Both these invocations will run once the worker is started
add.delay(1, 2)
multiply.delay(1, 4)

serve(add, multiply) # start a task worker listening for both `add` and `multiply`
```

### Run background tasks with multiple server replicas

For a highly available self-hosted deployment, configure every Prefect API server replica to use the same external Redis instance for Docket:

```bash
export PREFECT_SERVER_DOCKET_URL="redis://redis.example.com:6379/0"
```

Prefect uses Docket to retain a background task delivery until a task worker acknowledges it. If the API server that owns the task worker's WebSocket disconnects, another API server replica can reclaim and redeliver the task.

The default `memory://` Docket URL is process-local and does not provide durable or cross-replica delivery. Use a highly available Redis deployment when the Prefect server itself must be highly available. See the [`server.tasks.scheduling` settings](/v3/api-ref/settings-ref#servertasksschedulingsettings) to tune delivery concurrency and failover timeout.

Set task workers' `PREFECT_API_URL` to the load-balanced API endpoint, rather than an individual server replica, so state updates and subscription reconnects survive replica failure.

<Note>
Docket coordinates task delivery; it does not replace Prefect result storage. Task parameters and return values continue to use the task's configured Prefect result storage. Configure shared result storage when task submitters and task workers do not share a filesystem.
</Note>

### Submit multiple background tasks

To submit multiple background tasks, call `.map()` on the task with `deferred=True`.

{/* pmd-metadata: notest */}
```python
from prefect import task

@task(log_prints=True)
def add(a: int, b: int):
    print(f"{a} + {b} = {a + b}")

A = [1, 2, 3]
B = [4, 5, 6]

add.map(A, B, deferred=True)

add.serve()
```

<Tip>
`.map()` accepts `Iterable[P.args]`, `Iterable[P.kwargs]` or `unmapped` inputs.
</Tip>

If you want to keep one argument mapped and the other unmapped, you can do so by passing `unmapped` to the mapped argument.

{/* pmd-metadata: notest */}
```python
from prefect import task, unmapped

@task(log_prints=True)
def add(a: int, b: int):
    print(f"{a} + {b} = {a + b}")

A = [1, 2, 3]

add.map(A, unmapped(4), deferred=True)

add.serve()
```
