When you spawn subprocesses inside a flow or task — for example, with `multiprocessing.Pool`
or `concurrent.futures.ProcessPoolExecutor` — the Prefect run context is not automatically
available in the child process. This means `get_run_logger()` raises a `MissingContextError`
and logs are not sent to the Prefect API.

Use `with_context` from `prefect.context` to propagate the current run context into
subprocess workers. Logs emitted with `get_run_logger()` in the subprocess will be
associated with the parent flow run and task run and appear in the Prefect UI.

## `multiprocessing.Pool`

{/* pmd-metadata: notest */}
```python
import multiprocessing
from prefect import flow, task
from prefect.context import with_context
from prefect.logging import get_run_logger


def process_item(item):
    logger = get_run_logger()
    logger.info(f"Processing {item}")
    return item * 2


@task
def parallel_task(items):
    with multiprocessing.Pool() as pool:
        return pool.map(with_context(process_item), items)


@flow
def my_flow():
    results = parallel_task([1, 2, 3, 4])
    print(results)  # [2, 4, 6, 8]
```

## `concurrent.futures.ProcessPoolExecutor`

{/* pmd-metadata: notest */}
```python
from concurrent.futures import ProcessPoolExecutor
from prefect import flow, task
from prefect.context import with_context
from prefect.logging import get_run_logger


def process_item(item):
    logger = get_run_logger()
    logger.info(f"Processing {item}")
    return item * 2


@task
def parallel_task(items):
    wrapped = with_context(process_item)
    with ProcessPoolExecutor() as executor:
        return list(executor.map(wrapped, items))


@flow
def my_flow():
    results = parallel_task([1, 2, 3, 4])
    print(results)  # [2, 4, 6, 8]
```

## `multiprocessing.Process`

{/* pmd-metadata: notest */}
```python
import multiprocessing
from prefect import flow, task
from prefect.context import with_context
from prefect.logging import get_run_logger


def worker(result_queue):
    logger = get_run_logger()
    logger.info("Working in subprocess")
    result_queue.put("done")


@task
def spawn_process():
    q = multiprocessing.Queue()
    p = multiprocessing.Process(target=with_context(worker), args=(q,))
    p.start()
    p.join()
    return q.get()


@flow
def my_flow():
    result = spawn_process()
    print(result)  # "done"
```

## How it works

`with_context` captures the current Prefect run context (flow run, task run, settings, tags)
at the call site and returns a picklable wrapper. When the wrapper is called in a child process,
it restores the context before invoking your function. This enables `get_run_logger()` and
other context-dependent APIs to work as expected.

<Note>
The wrapped function must be defined at the module level (not inside another function)
so that it can be pickled by `multiprocessing`.
</Note>
