The Prefect Docs MCP server provides AI assistants with direct access to the official Prefect documentation through the [Model Context Protocol (MCP)](https://modelcontextprotocol.io/). This enables your AI assistant to answer questions about Prefect concepts, APIs, and best practices using up-to-date documentation—all without leaving your development environment.

## What is the Prefect Docs MCP server?

The Prefect Docs MCP server is an [MCP](https://modelcontextprotocol.io/) server that gives AI assistants the ability to:

- **Search documentation**: Find relevant Prefect documentation based on natural language queries
- **Access current content**: Get up-to-date information from the official Prefect docs
- **Answer questions**: Receive contextual answers about Prefect concepts, APIs, and workflows
- **Find examples**: Discover code examples and best practices from the documentation

Unlike the Prefect MCP server which connects to your Prefect instance, the Docs MCP server focuses solely on documentation access—no credentials or Prefect instance required.

## Prerequisites

Before installing the Prefect Docs MCP server, ensure you have:

- **Python 3.9 or newer** (for `uvx`)
- **uv** installed (Python package installer): `pip install uv` or `curl -LsSf https://astral.sh/uv/install.sh | sh`

No Prefect credentials or instance required—this server only accesses public documentation.

## Installation

### Local installation (stdio transport)

The default setup runs the MCP server locally using `uvx`:

```bash
uvx --from prefect-docs-mcp prefect-docs-mcp-server
```

No credentials or configuration required—the server accesses public documentation only.

### Cloud deployment (HTTP transport)

Deploy the MCP server to [FastMCP Cloud](https://fastmcp.cloud) for remote access from any client:

<Steps>
  <Step title="Fork the repository">
    Fork the [prefect-docs-mcp-server repository](https://github.com/PrefectHQ/prefect-docs-mcp-server) on GitHub
  </Step>

  <Step title="Deploy to FastMCP Cloud">
    1. Sign in to [fastmcp.cloud](https://fastmcp.cloud)
    2. Create a new server pointing to your fork
    3. Configure:
       - Server path: `src/prefect_docs_mcp_server/server.py`
       - Requirements: `pyproject.toml` (or leave blank)
  </Step>

  <Step title="Get your server URL">
    Copy the generated URL (e.g., `https://your-server-name.fastmcp.app/mcp`) and use it in your client configuration
  </Step>
</Steps>

<Note>
No environment variables or credentials are needed—the Docs MCP server only accesses public Prefect documentation.
</Note>

## Advanced configuration

### Configuration file locations

MCP server configurations are typically stored in these locations:

| Client | Configuration Location |
|--------|----------------------|
| Claude Code CLI | `~/.claude/config.json` |
| Cursor | `.cursor/mcp.json` (project-specific) |
| VS Code (Copilot) | `settings.json` (user or workspace) |
| Cline | `.cline/mcp.json` (project-specific) |
| Codex CLI | `~/.codex/config.toml` |
| Gemini CLI | `~/.gemini/settings.json` |
| JetBrains | IDE settings (varies by IDE) |

### Multiple server scopes

Some clients (like Claude Code) support different scopes for MCP servers:

- `--scope user`: Available across all projects (recommended)
- `--scope project`: Only available in the current project directory

```bash
# User-scoped (available everywhere)
claude mcp add prefect-docs --scope user -- uvx --from prefect-docs-mcp prefect-docs-mcp-server

# Project-scoped (only in current directory)
claude mcp add prefect-docs --scope project -- uvx --from prefect-docs-mcp prefect-docs-mcp-server
```

### Troubleshooting installation

**Server not connecting:**

1. Verify `uv` is installed: `uv --version`
2. Test the server manually: `uvx --from prefect-docs-mcp prefect-docs-mcp-server`
3. Review client logs (location varies by client)

**Command not found errors:**

Ensure `uvx` is in your PATH. Install `uv` if needed:

```bash
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Via pip
pip install uv
```

**Permission errors:**

Some clients may need explicit permissions to run external commands. Check your client's security settings and allowlists.

## Using both MCP servers together

The Prefect Docs MCP server and the Prefect MCP server complement each other:

- **Prefect Docs MCP Server**: Use for learning, documentation lookup, and finding examples
- **Prefect MCP Server**: Use for monitoring your instance, debugging runs, and querying your infrastructure

Install both to get the full AI-assisted Prefect experience:

```bash
# Claude Code marketplace installation
/plugin marketplace add prefecthq/prefect-docs-mcp-server
/plugin marketplace add prefecthq/prefect-mcp-server
/plugin install prefect-docs
/plugin install prefect
```

With both installed, you can seamlessly transition from learning about a concept (Docs server) to implementing it in your instance (MCP server).
